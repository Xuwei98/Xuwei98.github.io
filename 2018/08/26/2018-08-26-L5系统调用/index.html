操作系统基础 L5 系统接口的调用

发表于 2018-08-26 | 分类于 操作系统 | |
字数统计: 863 | 阅读时长 ≈ 3

重点知识章节

CPL、DPL、RPL说明

系统调用的直观实现

  1. 用户程序不能直接访问内核。
  2. 不能随意的调用数据,不能随意的jmp。

内核(用户）态,内核（用户段）

  • 内核（用户）段的内存执行在内核（用户）态下
  • 用户态不能跳跃到内核态执行

特权级（一种“硬件设计”）

  • 这种保护级成为保护环
  • DPL
    Descriptor Privilege Level
    目标段特权级
    被系统初始化。
    • DPL在GDT中,L3中提到,可翻阅前文获知。
      代码段,数据段的GDT表中对应的DPL都被head.s初始化为0.
  • CPL
    Current Privil Level
    当前特权级
    CPL存在CS和SS的第0位和第1位上。
    用户层执行时,CPL就为3.
  • 当mov,jmp只有CPL<=DPL,能进行访问。0是内核态,3是用户态。

硬件提供进入内核的方法

  • 中断指令int是进入内核的唯一方法

系统调用的核心

    1. 用户程序包含一段int指令的代码。
      根据用户的程序,执行的不同的int指令,跳去相应的地方执行中断函数。
  1. 举例,系统调用的实现。
    （应用程序）调用printf–>（C函数库）[库函数printf–>库函数write()]–>(OS内核)系统调用write()

库函数格式化,包含缓冲区,字符个数。然后调用write变成带int 0x80的内嵌代码,再通过系统调用进入到内核里。

write函数
  1. 用宏展开,替换参数。

  2. “=a”(res):”,输出。程序最后，将eax置给__res，返回给应用程序。
    “”(NR_##name),输入,mov _NR_write(系统调用号）,%eax。

  3. 把fd只给ebx,buf–>ecx,count–>edx。

  4. 执行int 0x80.
    进入内核。
  • 显然,set_system_gate用来设置0x80的中断处理


通过段选择符和偏移地址设置新的CS:IP = 8:(&system_call)
这时候CPL=0,因为CS为0x8,最后两位为00.已经跳到内核。

  1. 将结果保存到eax,将eax置给__res。
  2. 返回__res的值。
system_call

  • (sys_call_table+4*%eax)就是相应系统调用处理函数入口
  • 函数调用栈是4字节大小。
  • 第四个就是sys_write
  • call_sys_call_table(,%eax,4)就是call sys_write

如何找到字符串并传递给sys_write函数的呢

  • 通过寄存器找到用户态内存区中该字符串的起始地址，然后利用FS段寄存器找到并读入的

总结

用户程序–> printf展成 int 0x80 –> 中断处理system_call –>查表system_call_table –> _NR_write = 4 –>sys_write调用
